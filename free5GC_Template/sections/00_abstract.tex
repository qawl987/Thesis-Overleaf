Network slicing requires coordinated control across the radio access network (RAN), the 5G core, and the cloud-native infrastructure that hosts network functions. Existing open-source 5G deployments expose these controls through heterogeneous and low-level configuration interfaces, making it difficult for an operator to express service objectives such as high throughput or low latency as a single end-to-end policy. This paper presents an intent-driven orchestration system for free5GC and OCUDU that translates service-level slice intents into concrete core and RAN configurations. The system models intents as Kubernetes custom resources, maps intent expectations to 5QI, S-NSSAI, PRB ratio, and scheduling priority parameters, and applies the resulting configuration through a hybrid southbound design: Nephio/GitOps package mutation for RAN configuration and direct core-domain configuration for free5GC. A prototype implemented with Golang and Kubebuilder demonstrates two slices, eMBB and URLLC, over emulated UEs. Experiments show that the orchestration path consistently materializes slice configuration and that RAN scheduling parameters steer traffic differentiation between eMBB and URLLC. The measured management-plane latency is about 30 seconds, dominated by GitOps synchronization, while the local operator reconciliation step completes within a fraction of a second.
